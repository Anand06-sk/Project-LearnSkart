CS3551 - Distributed Computing Important Questions (Unit-wise)

Distributed systems and cloud computing are important for building modern applications that are scalable, reliable, and available across multiple machines. Studying important questions unit-wise helps students understand both core theory and algorithm-based concepts clearly for university exams.

UNIT I

This unit covers the basics of distributed models and computation structure. It helps students learn system models, shared memory ideas, and common design challenges in distributed systems.

  • Passing S/M vs Shared Memory S/M **
  • Model of distributed computations, global state
  • Designing issues, challenges and examples of DS **

UNIT II

This unit focuses on event ordering and consistent state capture in distributed environments. These concepts are useful for understanding synchronization behavior and message coordination.

  • CO (Causal Order) – full topic **
  • Message ordering paradigms
  • Snapshot algorithms for FIFO channels **

UNIT III

This unit introduces synchronization algorithms for mutual exclusion in distributed systems. Learning these algorithms improves clarity in algorithm-based exam questions and system-level reasoning.

  • Ricart–Agrawala algorithm, Maekawa’s algorithm
  • Lamport’s algorithm
  • Knapp’s classification (may be Part C)

UNIT IV

This unit explains recovery mechanisms in fault-prone distributed environments. Topics include checkpointing and rollback methods that are critical for reliable failure recovery.

  • Issues in failure recovery, Checkpoint-based recovery
  • Algorithm for asynchronous checkpointing and recovery
  • Log-based rollback recovery

UNIT V

This unit connects distributed computing with cloud platforms and services. It highlights cloud deployment models, virtualization, and service components used in practical cloud environments.

  • Characteristics of CC deployment models **
  • Virtualization, monitoring and cloud service platform
  • Remote storage, app services

Exam Preparation Tips

  • Prepare one unit per session and revise definitions before algorithms.
  • Practice algorithm flow for Lamport, Ricart-Agrawala, and recovery methods on paper.
  • Give special attention to distributed models, synchronization algorithms, recovery mechanisms, and cloud platforms.
  • Write short comparison notes for better recall in Part B and Part C answers.
